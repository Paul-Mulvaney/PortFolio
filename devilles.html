Paul Mulvaney

UX / UI Researcher, Designer & Developer

De'Villes

User Experince/ User Interface Design

Overview

To provide a rich online experience to entTice people to dine at the restaurant; to attract people to work for the restaurant, to provide an increase in the purchase of gift vouchers and to provide a intuative table booking facility, negating the need for phone bookings.

Design Keywords

Modern; Sophisticated; Fine-Dining; Atmospheric; Effortless

Empathy & Problem Definition

As part of understanding how DeVilles website can be improved one of the main areas that need to be focused on is looking at the existing site and seeing how thing exist in their current state. By doing this we can identify the strengths and weaknesses of the site. From here the problem that exists can be defined and then improved.

User Stories

To start understanding how to improve the site, the first thing that must be done is to decide how users are using the site. To do this user stories are created.hese are brief ideas of differant demographics that will be using the site, and how they will be using it. These user stories are made to represent 99% of user interactions on the site. By focusing the design for them, 99% of user interactions on the site should be refined for purpose.

Research

Based on the user stories a series of tasks is created for what of visitors to the site might be trying to do. These were: Making a reservation; Looking at the menu; Buying a gift voucher; Applying for a job. Using these as critia let me focus what I was looking to do on the site and potentially discover any areas potential users may be having problems. I attempted these tasks on the existing devilles site aswell as two competitors to understand the strenghts and weaknesses of each.

User Testing

To gather further data of the strenghts and weaknesses of these sites. The same tasks were preformed by multiple users respresenting potential visitors to the site. These users preformed these tasks on all three sites and their behaviours was recorded and noted for later use. For testing I used a sample user base of four people.

Empathy Mapping

With so much data gathered for all these tests, the next step is to organise the data and make sense of it. To do this an empathy map is created. This will contain what the user says, thinks does and feels.

Affinty Diagram & User Pain Points

From the empathy map. Similar points are grouped together into an affinty diagram. These points will become the user pain points, these are the most important areas of improvment for the new site. These new pain points are: 1. User needs to scroll too much 2. No clear “Book a Table” option 3. All content on one page 4. Cluttered Screen 5. Ugly and outdated Design

User Goals

With the user Pain points established, we have a clear idea of the problems the user is having. So fixing the issues is simple. Just do the opposite if the pain points. This is now the user goals. For this project the user goals are: 1. Eliminate need for excessive scrolling 2. Add a clear to see and use “Book a table” option 3. Divide the content over multiple webpages 4. Streamline the User Interface and remove unnecessary clutter 5. Create a modern and appealing design for the site. These goals will help guide the design in future steps.

Low Fidelity Design

Story Boarding

Working from the reserach established in earlier phases, the first stages of the Lo-Fi phase is storyboarding. This puts the designer in the mind of the customer and along with the user goals from before helps to established the ideal way the user will interact with the product.

Lo-Fi Prototyping

Following the story board and the users goals, Lo-Fi prototypes are made. These are basic pen and paper prototypes made using Marvel. This serves as a quick way to show a design to a potential user an idea and recieve feedback quickly without a big time or financial investment.

Lo-Fi Testing

With the Lo-Fi prototype made, the next thing to do is to show it to users. This is the most important step up to this point , as this is a user focused process. This is the application of all the research up to this point. Tests were recorded and behaviours, thought and suggestions were noted.

Lo-Fi Iterations

After test have been done. Like with research phases before it user feedback is noted and organised. Based on this user feedback, changes are made to the prototype which can be carried into the next stages of the design process.

Mid Fidelity Design

Mid-Fi Prototyping

With adjustments made and tested during Lo-fi prototyping the next step is Medium Fidelity prototyping. These are a more evolved version of the Lo-Fi prototype but features a more true to life UI. However this will not feature any images or colours to keep the user focused on the taask they are trying to proform with out distraction.

Mid-Fi Testing

Just Like the Lo-Fi Prototype before ther Mid-Fi Prototypes need to be tested by a group of users. Again same as before these user tests are recorded, with interactions and suggestions noted to contribute to further improvment and iterations.

Mid-Fi Iterations

With Mid-Fi testing finished suggestions from users tests are introduced and areas where users were having issues were streamlined and fixed. With all of these iterations completed and tested, the users interactions for the site have been refined and all that is left is to style the user interface.

High Fidelity Design

Hi-Fi Prototyping

The Last and one of the most interesting phase of the design process is Hi-Fi Prototyping. With the user interactions completed this phase is fully focus on aesthetics. This phase is built from the Mid-Fi Prototype and is created by choosing colours and adding elements, effects and images to make the site feel whole.

Defining The Visual Brand

While this can be the most enjoyable step it is also possibly the most challenging. One miss step and the design falls apart. That's why at this point it is very important to take great care to cultivate a mood while designing the site. For this design I wanted to cultivatea mood on the site to match that of a the feeling of being inside the restaurant for a romantic candle lit dinner.

A/B Testing

An often overlooked step during this phase is user input. While the User interactions have been tested fully the design has not been a focus until now. It is important to take the time to keep the users involved and get their design preferances through A/B Testing.
