Work

Brief from client

After recieving the brief from the client the next steps are to define the problem. Once I have a problem defined I can begin to craft a solution, and once I know what the solution will look like, I can take steps to make that solution a reality

Problem Definition

To start I thouroughly read the brief, to fully understand the client needs. Next I will tell the client what needs to be done to meet their needs, so they can understand the timeframe of events. Next I will establish sucess criteria which will give quantative results for the finised product

Secondary Research

In this phase of the process I will set myself tasks to achieve using the clients existing website. I make notes of these opservation and note what is working and what needs to be improved.

User Stories

User stories are created. These are brief ideas of differant demographics that will be using the site, and how they will be using it. These user stories are made to represent 99% of user interactions on the site. By focusing the desing for them, 99% of user interactions on the site should be refined and pleasurable

Usability Testing

While I have tested the website and noted my issues. This is user-experience. Not my-experience. So I gathed a group of users who best represented my user stories and set them the same tasks I have done to see how they will perform with them.

Empathy Mapping

With all the finding from the Usability test noted. These are written on post it notes and are categorised into what the user "Does"; "Feels"; "Thinks" and "Says". With reoccuring notes being grouped together.

User Pain Points

With the Empathy Map completed. Similar post-its are grouped togther. The biggest clusters are the areas that are most in need of improvment. These become my User Pain Points.

User Goals

With the pain points created. It's time to take the greatest weaknesses and turn them into the greatest strengths. e.g. If the users think a particular interaction took too long. The focus is now to make that process quick and easy.

Journey MAp

The last step here is to make a Journey Map. This will show the users emotional state as they progress through a task. I want to make sure that where the user is unhappy, with a finshed design I want them be be thrilled.

With all the reseach conducted. I now need to refine that reseach into a solution for the users and the client.

Ideal Customer Journey

As I now have a visual representation of what the current customer journey looks like. Why not have one of what I want the customer journey to be. That way I can see what the user should be feeling as they interact with the website, and design so these expectations are met.

Story Board

The next stage of the process is to take the ideal customer Journey Map and create a story board. This will show me visually what the user is doing, where they're doing it and why. This might seem silly and un-necessary to have both but creating a clear picture of the user is maybe some of the most important time spent on any project.

Lo-Fi Prototyping

With an clear visual idea of what a finsihed user journey will look like, I can now begin to design a prototype that will meet achieve this user journey. The quickest and easiest way to do this? With a UX designers best friend. Post-it notes. With prototypes entire user interactions can be made in minutes.

Lo-Fi Usability Testing

With the quick pen and paper Low-Fidelity prototype made, what is the most important part in User Experience. To show it to users! By getting the users back to test the new prototype and give feedback.

Lo-Fi Iteration

Based on feeback from users I can then take the Lo-Fi, keep what works, get rid of whats not needed and improve on whats not working right. Thus, a finished design is one step closer.

Try Out My Lo-Fi Prototype

With the Lo-Fi Iterations made I can now move onto Medium Fidelity. Using Adobe XD I can quickly create interactive prototypes for users to test and give fedback on.

Mid-Fi Prototyping

As the hard work of defining the user flow has been done in the lo-Fi stage, Mid-fi is quite quick and easy. All I need to do is follow the blueprint from my Lo-Fi prototype and make this look more like a tradition website. This prototype is made in grayscale so when tested by users they will not be distracted by colours or images.

Mid-Fi Usability Testing

I can now test this prototpye on users to see how they like it. After all whats User Experience with out users?

Mid-Fi Iteration

Just like the Lo-Fi stage I can take the feedback from users gathered from users. Iterate the design based on this feedback, and create an een more polished design.

Try Out My Mid-Fi Prototype

Now the real fun starts! After Lo-Fi and Mid-Fi testing and iterations have worked out the kinks in the user flow and interactions, in High-Fidelity, all that's left is to give this site a little face lift to getting looking production ready!

Hi-Fi Prototyping

This is maybe the most fun part of the process. No interactions or flows need to be worked out. So what I can do is take my mid-fi prototype and start adding elements, effects and images to make the site feel whole.

Defining the Visual Brand

While this is the most fun step it is also possibly the most challenging. One miss step and the design falls apart. That's why at this point I took great care to cultivate a mood while designing the site. I wanted the feeling of the site to match that of a the feeling of being inside the restaurant for a romantic candle lit dinner.

A/B Testing

As Ive said before this is my experience. This is UX. It would be foolish of me to make every decision and think it was perfect. So, whenever a big design change or something that moved away from expected conventions of web design was introduced, A/B tests were conducted. A group would be shown multiple designs and asked their favourite. These results were polled and the winning design was the one that made it into the finshed design.
