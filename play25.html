Paul Mulvaney

User Experience / User Interface Designer

Play 25 Online

User Experince/ User Interface Design

Overview

To build the ultimate home for the beloved Irish card game 25. An online home for the game, perfect for beginners and veterans alike. A haven where beginners can learn to play their first trick and a place where veterans can meet up to keep their skills sharp.

Design Keywords

Fun; Quick; Engaging; Addictive;

Empathy & Problem Definition

User Stories

To start understanding how to building the game, the first step is to look at the users who will be playing the game and when and where they will be playing it. This will help to focus the design process by honing in the potential user base and how they want to play. Instead of trying to build something for everyone. We will use these stories to build task for reseach and user testing later.

Learning to Play

Unlike other User Experience projects this one requires a lot of work on the part of the designer before starting. An important phase of this project was to learn how to play the game physically. It would be impossible to try and design a game without first knowing how to play it.This will also help further understand the user base and what they enjoy about the game.

Research

Having learned how to play the game the next phase is to follow the tasks based on our user stories. For this project tasks were: 1. Set up private room to play with family; 2. Set up an offline Game; 3. Learn to play Game. These task were prefered on the current offering from play 25 aswell as some competitor sites that exist. Observations, strenghts, weaknesses and any issues were noted for later use.

User Testing

To gather further data of the strenghts and weaknesses of these sites. The same tasks were preformed by multiple users respresenting potential visitors to the site. These users preformed these tasks on all three sites and their behaviours were noted for later use. For testing I used a sample user base of five people.

Empathy Mapping

With so much data gathered for all these tests, the next step is to organise the data and make sense of it. To do this an empathy map is created. This will contain what the user says, thinks, does and feels.

Affinty Diagram & User Pain Points

From the empathy map. Similar points are grouped together into an affinty diagram. These points will become the user pain points, these are the most important areas of improvment for the new site. These new pain points are: 1. Lack of tutorials, a learning platform or help within the app 2. Game doesn't feel like a game. There is no real-world counterpart. 3. Lack of options for different game modes (Private, Offline) 4. Site feels unsafe or suspicious 5. User decisions have unclear outcomes 6. Users are getting frustrated and are leaving the site

User Goals

With the user Pain points established, we have a clear idea of the problems the user is having. So fixing the issues is simple. Just do the opposite if the pain points. This is now the user goals. For this project the user goals are: 1. Clear tutorial needs to be created, and help and learning platforms needs to be created 2. Gameplay needs to “feel” more like its real-world counterpart (real deck of cards etc.) 3. Further options for different game modes need to be created 4. Site needs to be modernised, spellchecked, and polished to make the user feel secure while playing 5. Labels on buttons and menus need to be clearer 6. Game needs to be made more inviting, with added tutorials and gameplay needs to be clarified for users. These goals will help guide the design in future steps.

Low Fidelity Design

Lo-Fi Prototyping

Looking at the User goals the most important for the future longevity of the game was the need for a tutorial. In initial reseach many of the users had never heard of the game prior. Thus it was decided to build the community for the app teaching them how to play quickly and effectivly was the most important aspect to focus on. Thus this was the focus of the lofi prototype. A simple three phase tutorial was made using marvel app which would teach users all the basics of the game and encourage them to play more.

Lo-Fi Testing

With the Lo-Fi prototype made, the next thing to do is to show it to users. This is a very important phase. As we are testing the tutorial if this is not affective at teaching the user the process needs to starts over. For these tests the same user base was used as in the research phase. To make the tests feel more personal these test were conducted in person. Initailly the user was given the app to test. After this they were dealt a full hand of cards and asked to play. This would determine the sucess of the tutorial.

Lo-Fi Iterations

After tests have been done and determined to be a sucess in teaching the user the game, user feedback is noted and organised. Based on this user feedback, changes are made to the prototype which can be carried into the next stages of the design process. For this project users responded very positvly to encouragement and farmiliar language, this was increased to make the user feel more welcome and at ease.

Mid Fidelity Design

Mid-Fi Prototyping

With adjustments made and tested during Lo-fi prototyping the next step is Medium Fidelity prototyping. These are a more evolved version of the Lo-Fi prototype but features a more true to life UI. However this will not feature any images or colours to keep the user focused on the taask they are trying to proform with out distraction. This however will show the user how the game will feel and play and thus is very important to make sure it is correct.

Mid-Fi Testing

Just like the Lo-Fi Prototype before ther Mid-Fi Prototypes need to be tested by a group of users. As this is a game tutorial and not an app or webpage, tests need to be treated differently. You cant teach people to play the same game twice. Thus for this test new users needed to be brought in. Only one extra user was brought to again test the quality of the tutorial and ensure users still learned to play. However the same user base from previous tests was used to test game feel and the UI.

Mid-Fi Iterations

Just as before any observations, strengths, weakness and suggestions from users were noted and used for alterations and improvments. These changes were then tested again to ensure the quality, and brought into the next steps of development.

High Fidelity Design

Hi-Fi Prototyping

The Last and one of the most interesting phase of the design process is Hi-Fi Prototyping. With the user interactions completed this phase is fully focus on aesthetics. This phase is built from the Mid-Fi Prototype and is created by choosing colours and adding elements, effects and images to make the site feel whole.

Defining The Visual Identity

One of the more challenging steps in the design process was finding the visual identity of the app. The goals outlined by the client were to create a welcoming place that contained a strong sense of irish culture. To find a balance that was welcoming for an internation audience but not cheesy or pandering to an irish audience was one of the more challenging aspects of the design process.

A/B Testing

To ensure that the design process never moves away from a user centered process A/B tests were conducted regularly. This meant that user input was always considered in the design process and that the design was always built with the users in mind first and foremost.
