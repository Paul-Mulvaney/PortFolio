Paul Mulvaney

UX / UI Researcher, Designer & Developer

Play 25

User Experince/ User Interface Design/ Game Design

Role: Sole UX Designer/Researcher

Platform: Mobile

Timeframe: 12 Weeks

Solution

To build the ultimate home for the beloved Irish card game 25. An online home for the game, perfect for beginners and veterans alike. A haven where beginners can learn to play their first trick and a place where veterans can meet up to keep their skills sharp.

01.

Research

To begin to create a place to entice players to join an understanding of how what existing players want from a digital version of the game, what brings them back playing the real thing and what made them fall in love with the game initially needs to be established.

Research

Initial research on the history of the game and peoples firmiliarity of the game returned a similar theme, people either have never heard of the game or havn't played it in years. Therefore it was decided early on in the design process that the focus of the project would be to creat a detailed tutorial to teach and introduce the game to new players

Learning to play

The first step is to learn to play the game and have a clear understaning of how it functions. Doing so helps to better understand the game to help the design but also acts as research in finding difficulties in the way people learn how to play.

Teaching Others & Testing

Now that I can play the game the next step is to teach others. This again acts as research for how people learn and interact the game but also to gain insight on difficulties people have when learning the game. These people were then tested using the exiting site for their feedback.

Empathy Maps

All pain points from teaching the game and from the users interacting with the website were noted and organised on a Empathy Map.

Pain Points

  • Lack of tutorials, a learning platform or help within the app
  • Game doesn't feel like a game. There is no real-world counterpart
  • Lack of options for different game modes (Private, Offline)
  • Site feels unsafe or suspicious
  • User decisions have unclear outcomes
  • Users are getting frustrated and are leaving the site

User Goals

  • Clear tutorial needs to be created, and help and learning platforms needs to be created
  • Gameplay needs to “feel” more like its real-world counterpart (real deck of cards etc.)
  • Further options for different game modes need to be created
  • Site needs to be modernised, spellchecked, and polished to make the user feel secure while playing
  • Labels on buttons and menus need to be clearer
  • Game needs to be made more inviting, with added tutorials and gameplay needs to be clarified for users

02.

Prototyping, Testing & Iteration

The next phase of the design process involves creating and testing prototypes built upon the research gathered earlier. This will involve creation of prototypes to be shown to users for futher feedback and iterations. This Project more than most others required constant iteration and fine tuning as no digital tutorial for this game has been created before.

Usability Testing

Usability testing puts prototypes in front of users to gather valuable feedback which can be used to further improve the product.

Test Findings

Feedback gathered from usability tests can be gathered grouped and decided what parts of the product needs iteration.

Prototype Iterations

Based on feedback iterations can be made to furter guide the project to a more polished finsihed product.

03.

Finished Design

From a culmination of all research, testing and iterations the finshed high fidelity User interface can be designed and finalised.
From all research a three phase tutorial was decided that would gradually introduce rules to playes with out them feeling overwhelmed.

Visual Identity

Using various fonts, images, designs and icons a visual brand that represents the theme and feeling of the product.

A/B Testing

Multiple designs can be made and shown to users for feedback, based on this the finished design will be one that will be the most desirable.

Hi-Fi Design

Finished designs can be created from a cumilation of all feedback.
